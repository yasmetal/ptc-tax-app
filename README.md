# PTC Tax App — เครื่องมือคำนวณภาษีบริษัท/ห้างหุ้นส่วนนิติบุคคลไทย

เว็บแอป static (HTML + Vanilla JS) ใช้งานบนมือถือได้ รันบน GitHub Pages ไม่มี server —
ข้อมูลเก็บใน localStorage ของเบราว์เซอร์ + สำรองขึ้น Google Drive ของผู้ใช้เองได้

> ⚠️ **คำเตือน:** เครื่องมือช่วยคำนวณเบื้องต้นเท่านั้น ไม่ใช่คำแนะนำทางภาษีที่เป็นทางการ
> โปรดตรวจสอบกับนักบัญชี/ผู้สอบบัญชีรับอนุญาต และ [rd.go.th](https://www.rd.go.th) ก่อนยื่นแบบจริง

## ความสามารถ

| โมดูล | รายละเอียด |
|---|---|
| ภ.ง.ด.51 | ภาษีครึ่งรอบบัญชีจากประมาณการ + เตือนความเสี่ยงประมาณการขาดเกิน 25% (เงินเพิ่ม 20% ม.67 ตรี) |
| ภ.ง.ด.50 | ปรับกำไรบัญชี→ภาษี (รายจ่ายต้องห้าม ม.65 ตรี, ค่าเสื่อมอัตราสรรพากร, ขาดทุนยกมา ≤5 รอบบัญชี, สิทธิ SME) หักเครดิต ภ.ง.ด.51/WHT |
| SME | ตรวจเงื่อนไขอัตโนมัติ (ทุนชำระแล้ว ≤5 ล้าน **และ** รายได้ ≤30 ล้าน) → อัตราขั้นบันได ยกเว้น 300,000 / 15% / 20% |
| WHT | ทะเบียนรายธุรกรรม แยก ภ.ง.ด.1 (อัตราก้าวหน้า) / ภ.ง.ด.3 / ภ.ง.ด.53, พิมพ์หนังสือรับรอง 50 ทวิ เป็น PDF, ส่งออก CSV ใบแนบ |
| ภ.พ.30 | ใบกำกับภาษีรายใบ (ขาย/ซื้อ/ภาษีซื้อต้องห้าม) กระทบยอด + เครดิตยกไปเดือนถัดไปอัตโนมัติ |
| เบี้ยปรับ/เงินเพิ่ม | เงินเพิ่ม 1.5%/เดือน (เพดานเท่าภาษี), เบี้ยปรับ VAT ลดตาม ท.ป.81/2542 (2/5/10/20%), ค่าปรับอาญา |
| ปฏิทินภาษี | คำนวณ deadline จากรอบบัญชีของบริษัท (fiscal year ไม่ต้องตรงปีปฏิทิน) รวมสิทธิ e-Filing +8 วัน |

## อัตราภาษีเป็นข้อมูล ไม่ hardcode

อัตรา/เกณฑ์ทั้งหมดอยู่ใน `js/config.js` เป็นโครงสร้าง **versioned มี effectiveFrom/effectiveTo**
engine เลือกเวอร์ชันตามวันที่ธุรกรรม → คำนวณย้อนหลังถูกต้อง และเมื่อกฎหมายเปลี่ยน
(เช่น พ.ร.ฎ. ต่ออายุ VAT 7%) แก้ได้จากหน้า **ตั้งค่า** ในแอปโดยไม่ต้องแตะโค้ด

ข้อมูลตรวจสอบล่าสุด 21 ก.ค. 2569 จาก:
- อัตรา CIT/SME: <https://www.rd.go.th/841.html>
- VAT 7% (พ.ร.ฎ. ฉบับที่ 799 ถึง 30 ก.ย. 2569): <https://www.rd.go.th/35772.html>
- ลดเบี้ยปรับ ท.ป.81/2542: <https://www.rd.go.th/3361.html>
- รายจ่ายต้องห้าม ม.65 ตรี: <https://www.rd.go.th/827.html>
- ขยายเวลา e-Filing +8 วัน (ถึง 31 ม.ค. 2570): <https://efiling.rd.go.th/rd-cms/extend>

## การใช้งาน

เปิด `index.html` ตรง ๆ หรือใช้ผ่าน GitHub Pages ได้เลย (มือถือ/เดสก์ท็อป)

### ตั้งค่า Google Drive Sync (ครั้งเดียว)
1. ไป [Google Cloud Console](https://console.cloud.google.com) → สร้าง Project
2. เปิดใช้ **Google Drive API** (APIs & Services → Enable APIs)
3. OAuth consent screen → External → เพิ่มอีเมลตัวเองเป็น Test user
4. Credentials → Create Credentials → **OAuth client ID** → Web application
   - Authorized JavaScript origins: ใส่ URL ของ GitHub Pages เช่น `https://<user>.github.io`
5. คัดลอก Client ID มาวางในแอป หน้า **ตั้งค่า → Google Drive Sync**
6. กด Backup/Restore ได้เลย (ข้อมูลเก็บใน appDataFolder — โฟลเดอร์ซ่อนเฉพาะแอปใน Drive ของคุณ)

ไม่ตั้งค่า Drive ก็ใช้ **Export/Import JSON** สำรองไฟล์เองได้

### รันชุดทดสอบ
```bash
node tests/run-tests.js   # 57 test cases เทียบตัวอย่างจากเอกสารกรมสรรพากร
```

## ความปลอดภัย & PDPA
- ไม่มี server กลาง — ข้อมูลการเงินอยู่ในเครื่องผู้ใช้เท่านั้น (และ Drive ส่วนตัวหากเปิด sync)
- มี Audit log บันทึกการเปลี่ยนแปลงในแอป
- ข้อมูลบุคคลของคู่ค้า/พนักงาน (ชื่อ, เลขผู้เสียภาษี) อยู่ภายใต้ PDPA — จำกัดผู้เข้าถึงเครื่อง,
  ล็อกหน้าจอ, ลบข้อมูลเมื่อหมดความจำเป็น, และอย่าใช้บนเครื่องสาธารณะ
